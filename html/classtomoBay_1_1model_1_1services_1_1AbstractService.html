Detailed Description

Allows the system to only accept runnable's that implement this interface rather than all runnables.

As well as providing an interface that all abstract service should conform to.

There are two varieties of AbstractService Derived Type: an unconfigured Service and a configured service. for an unconfigured service ignore the setConfig method as the service will ignore it anyway.

for a configured service it is necessary to provide an AbstractConfiguration object in order for it to work properly
