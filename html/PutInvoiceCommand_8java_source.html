25 /**
26  * this class instructs winstock to create an invoice given the data that is passed in.
27  * The format of the message to create (as defined by winstock spec) is as follows:
28  *
29  * PutInvoiceMessage
30  * ==================
31  * | DataType | Label | Notes |
32  * |:---------:|:-------------:|:--------------------------:|
33  * |byte | Type | 34 |
34  * |byte | Company | 0=ford, 3=citroen, 8=fiat |
35  * |char | AccountCode[9]| account identifier |
36  * |char | ship1[51] | name |
37  * |char | ship2[51] | address line 1 |
38  * |char | ship3[51] | address line 2 |
39  * |char | ship4[51] | city |
40  * |char | ship5[51] | county |
41  * |char | Postcode[10] | postcode |
42  * |char | Order[21] | order data |
43  * |short | Lines | the # of invlines to follow|
44  * |InvLine | Il[1] | see below table |
45  *
46  * InvLine
47  * ========
48  * | DataType | Label | Notes |
49  * |:---------:|:-------------:|:-------------------------:|
50  * |char |part[17] |the part number |
51  * |char |Description[31]|the description of the part|
52  * |int |Quantity | |
53  * |int |Price | |
54  * |int |InStock | |
55  *
56  * **nb//the numbers within square brackets are the maximum length that this field can length**
57  *
58  * this class is a concrete command in a Gof style command pattern
59  *
60  * @author Jan P.C. Hanson
61  * @see https://dzone.com/articles/design-patterns-command
62  *
63  */
